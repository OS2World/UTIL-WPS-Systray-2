 1. Что это такое?

 Systray это  то,  с чем многим из нас приходится иметь дело
 при работе с операционными системами Windows '95/98/NT.  На
 самом  деле  кроме  банального  таскбара это еще и довольно
 удобное средство, называемое трэем. Трей представляет собой
 небольшой   участок   экрана,  отведенный  для  программ  в
 настоящий момент не находящихся на экране,  но  активных  и
 выполняющих  какие-либо действия который пользователь может
 проконтролировать через этот же  самый  трей.  От  обычного
 таскбара его отличают несколько вещей:

 Программа,  имеющая  свою  иконку  на  трэе, самостоятельно 
 управляет    этой     иконкой.    Действия,    производимые  
 пользователем  над  иконкой программы на трэе, пересылаются
 как сообщения этой программе и  управляют ходом  ее  работы
 через  всплывающие  меню.  Примеры: Антивирусы,   регулятор   
 уровня   звука,   индикатор   и   переключатель   раскладки 
 клавиатуры.    Программа     получает   удобный    механизм
 информирования пользователя о ходе выполнения своей работы. 
 Примеры: ReGet, ICQ, The Bat.



 2. Описание программы.


 2.1. Установка

 Для установки программы запустите скрипт install.cmd, 
 который создаст папку Systray/2, объекты программы и 
 документации а также тень Systray/2 в папке Startup.
 После этого, Вы можете запустить объект Systray с 
 настройками по-умолчанию (файл systray2.ini). 

 Внимание: если Вы запускаете Systray не в виде объекта,
 а непосредственно EXE-файл, то либо Вы должны указать
 в качестве параметра INI-файл конфигурации (systray2.ini
 или любой другой, если он не существует - будет создан).
 Если в качестве параметра ничего не задано, по умолчанию
 будет использоваться системный файл OS2.INI.


 2.2. Настройка и возможности 

 Основные пункты настройки Systray не должны вызывать
 вопросов и проблем и будут полностью описаны в конечной
 документации, здесь же будет кратко описаны настройки и 
 состав базового плагина Systray.dll

 Внутри плагины состоят из модулей (Unit), которые 
 являются отдельными функционально законченными программами,
 способными выполняться под управлением Systray/2.

 Стандартный плагин включает в себя:

SystrayClockClass  	отображает текущее время, в настройках
			можно определить:
			- отображение секунд;
			- отображение даты;
			- режим 12-часового формата времени.

			цвет фона окна часов и шрифт, которым
			выводится время могут быть изменены
			путем drag-and-drop цветов и шрифтов
 			из соответствующих палитр OS/2.

SystrayToolbarClass	Toolbar, аналог Launchpad или Tray из WarpCenter.
			служит для настраиваемой полосы быстрого доступа
			к программам. В настройках этого модуля можно
			- определить программы, которые будут находиться
			  в Toolbar'е;
			- порядок их следования;
			- параметры запуска и рабочий каталог.

SystrayDiskMonitor	Монитор свободного места на диске

SystraySwapMonitorClass Монитор файла подкачки swapper.dat

SystrayClass		Системный трэй для отображения и управления
			запущенных процессов. 

SystrayTaskbarClass	Переключатель задач. Его настройки:
			- не показывать иконки процессов (только названия);
			- выбор вида кнопок задач - Warp4 или Warp3;
			- размер кнопок задач ("растягивание" на всю ширину).

			Кроме того, переключатель задач имеет фильтры и
			может скрывать отдельные процессы из списка задач.
			Эти фильтры настраиваются как из окна настроек
			этого модуля, так и по правому клику мыши на 
			кнопках процессов.

			Также, по правому клику на кнопке задачи, есть
			возможность прекращения ее работы (пункт "Close").

 
 Не рекомендуется создавать больше одного объекта SystrayClass или
 SystrayTaskbarClass.

 Можно запускать одновременно несколько копий systray.



 3. Tray API.

 В прилагаемом примере  trayex.exe  (при  веключенном  трее)
 показывается  использование  Tray  API  (пример  достаточно
 простой, реально с помощью Tray API уже можно делать много,

 В поставке  Systray/2  идут  прилагаются  исходные   тексты
 TrayEx'а и описание Tray API для использования системного
 трэя в ваших программах




 4. Известные проблемы

 Если Вы запускали версию Systray alpha 1 - удалите из системного
 реестра (OS2.INI) весь раздел HINI_USER_PROFILE\NewSystray (это
 можно сделать например, с помощью программы REGEDIT2). 



 5. Исходные тексты

 Исходные  тексты  примера  использования Tray API открыты и
 могут свободно использоваться за исключением применения в 
 коммерческих целях и без указания ссылки на проект Systray.
 
 Исходные тексты Systray будут открыты начиная с первой бета
 версии  (в виду  текущей  переработки  исходников на стадии 
 альфа-версий открытие их представляется неразумным).
